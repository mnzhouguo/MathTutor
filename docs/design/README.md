# UI/UX 设计规范

## 设计理念

### 简洁至上（Less is More）

- 去除一切不必要的视觉元素和装饰
- 每个页面只保留核心功能，避免信息过载
- 留白是设计的一部分，而非浪费空间
- 用户视线应该自然聚焦在重要内容上

### 情感化设计

- 使用温暖的色调，减少学习焦虑感
- 通过微交互（如按钮悬停、页面切换）传递友好感
- 避免冷冰冰的技术感，营造陪伴式学习氛围

## 色彩系统

### 主色调

- 主色：柔和的蓝色 `#4A90E2`（专业、冷静、可信赖）
- 辅助色：温暖橙色 `#FF9500`（活力、鼓励、提示）
- 成功色：清新绿色 `#34C759`（正确、完成、进步）
- 错误色：柔和红色 `#FF3B30`（温和提醒，而非严厉警告）

### 中性色

- 背景色：纯白 `#FFFFFF` / 浅灰 `#F5F5F7`（分层次）
- 文字色：深灰 `#1D1D1F` / 中灰 `#86868B`（建立视觉层级）
- 边框色：浅灰 `#E5E5EA`
- 阴影：`rgba(0, 0, 0, 0.08)`（极淡的阴影，营造层次）

### 色彩使用原则

- 同一页面不超过 3 种主要颜色
- 重要操作使用主色，次要使用中性色
- 避免高饱和度色彩，减少视觉疲劳

## 排版系统

### 字体选择

- 标题：PingFang SC / Helvetica Neue / Arial（无衬线，清晰现代）
- 正文：PingFang SC / Microsoft YaHei（中文友好）
- 数学公式：KaTeX / MathJax（专业渲染）

### 字号层级

- 大标题：24-28px（页面主标题）
- 中标题：18-20px（区块标题）
- 小标题：16px（卡片标题）
- 正文：14-16px（常规文字）
- 辅助文字：12-13px（次要信息）

### 行高与间距

- 正文行高：1.6-1.8（保证可读性）
- 标题行高：1.2-1.4（紧凑有力）
- 段落间距：16-24px
- 元素间距：8px 的倍数（8/16/24/32/48px）

## 布局原则

### 网格系统

- 采用 12 栏网格系统，保证响应式适配
- 内容最大宽度：1200px（超过则居中显示）
- 边距：桌面端 32px，移动端 16px

### 信息层级

- **F 型布局**：利用用户阅读习惯，重要信息放在左上
- **卡片式设计**：相关内容组织在卡片内，通过阴影和圆角区分
- **视觉权重**：通过大小、颜色、位置建立清晰的信息层级

### 响应式设计

- 移动优先（Mobile First）设计策略
- 断点：768px（平板）、1024px（桌面）
- 移动端：单列布局，简化操作
- 桌面端：多列布局，充分利用空间

## 组件设计规范

### 按钮

- 圆角：6-8px（友好但不失专业）
- 高度：44px（符合触控最小尺寸）
- 主按钮：填充主色，白色文字
- 次按钮：描边或浅灰色背景
- 悬停效果：轻微上浮 + 阴影加深（过渡动画 200ms）

### 输入框

- 圆角：6px
- 高度：44px（与按钮对齐）
- 边框：1px solid #E5E5EA
- 聚焦状态：边框变为主色，添加淡蓝色阴影
- 占位符：#86868B（低对比度）

### 卡片

- 圆角：12px（现代感）
- 阴影：`0 2px 8px rgba(0, 0, 0, 0.08)`
- 悬停效果：`0 4px 16px rgba(0, 0, 0, 0.12)`（过渡 300ms）
- 内边距：24px

### 图标

- 风格：线性图标（Line Icons），2px 线宽
- 尺寸：24px（标准）、32px（大图标）
- 颜色：继承文字颜色或使用主色

## 交互设计原则

### 即时反馈

- 所有交互操作都应有视觉反馈（按钮点击、输入框聚焦等）
- 加载状态使用骨架屏（Skeleton Screen）而非转圈动画
- 操作成功使用 Toast 提示（2-3秒自动消失）

### 微交互动画

- 过渡时间：200-300ms（快速但可见）
- 缓动函数：ease-out（自然减速）
- 动画类型：透明度、位移、缩放（避免复杂路径）

### 减少认知负担

- 隐藏次要功能，按需显示（渐进式披露）
- 使用默认值，减少用户决策
- 错误提示提供具体解决方案，而非技术术语

## 无障碍设计

- 色彩对比度符合 WCAG AA 标准（至少 4.5:1）
- 所有交互元素可通过键盘访问
- 为图片和图标提供 alt 文本
- 支持字体缩放（最高 200%）
