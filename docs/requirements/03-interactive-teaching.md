# 3. 互动式1对1智能教学

## 功能定位
核心教学场景,基于"专题→题目→逻辑链步骤"的渐进式教学路径,AI教师扮演苏格拉底式导师角色。

## 3.1 教学流程概述

### 核心理念
通过结构化的支架式提示引导学生主动思考,而非直接给出答案。

### 完整教学流程

```mermaid
学生选择专题
  ↓
系统加载专题相关题目（按难度排序）
  ↓
选择第一题开始学习
  ↓
进入题目逻辑链步骤教学
  ↓
┌─────────────────────────────────┐
│ 当前步骤：展示题目和目标         │
│ ↓                                │
│ 学生尝试回答（输入/语音）         │
│ ↓                                │
│ AI判断答案正确性                 │
│ ├─ 正确 → 进入下一步骤           │
│ └─ 错误 → 提供支架提示（1-3级）  │
│     ↓                             │
│   学生根据提示重新思考           │
│     ↓                             │
│   AI继续引导（最多3轮）         │
│     ↓                             │
│   仍不会 → 调用实验工具演示     │
│     ↓                             │
│   再次尝试或查看答案解析        │
└─────────────────────────────────┘
  ↓
完成所有步骤 → 题目学习总结
  ↓
进入下一题
  ↓
完成专题所有题目 → 专题学习报告
```

## 3.4 支架式提示策略

### 三级提示体系

**Level 1 - 观察提示**
```
"再仔细读一下题目，M和N的起点在哪里？速度分别是多少？
它们是同向运动还是相向运动？"
```

**Level 2 - 方法提示**
```
"追及问题中，相遇时两者走过的路程之和等于什么？
还记得追及问题的基本公式吗？"
```

**Level 3 - 具体步骤提示**
```
"设运动时间为t秒：
- M从A出发走的路程 = 2t
- N从B出发走的路程 = 3t
- 相遇时：2t + 3t = AB的距离
试试列出方程。"
```

### AI教学策略

1. **第一轮**：给出观察提示，等待学生思考
2. **第二轮**：如果仍不会，给出方法提示
3. **第三轮**：如果仍不会，给出具体步骤提示
4. **第四轮**：调用实验工具进行可视化演示
5. **最后**：如果仍然不理解，提供详细答案解析

## 3.7 交互设计要求

### 实时反馈
- 学生输入后立即给予反馈（< 500ms）
- 使用进度条显示当前步骤位置
- 答案正确时使用积极反馈动画

### 多模态输入
- 支持文字输入
- 支持语音输入（自动转文字）
- 支持图片上传（手写答案识别）

### 个性化体验
- AI教师语音风格可选（温柔/活泼/严谨）
- 界面主题可选（明亮/暗黑/护眼）
- 提示详细程度可选（精简/详细/超详细）

## 3.8 技术实现要点

- 使用WebSocket保持实时通信
- 语音识别集成第三方SDK（如讯飞、百度）
- 手写识别使用OCR技术
- AI对话使用大语言模型API
- 学习进度使用数据库持久化存储
